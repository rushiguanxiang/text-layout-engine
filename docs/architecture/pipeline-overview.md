# 流水线总览图

## 架构总览

TextLayoutEngine 的核心是一条从数据输入到像素输出的、无状态的纯函数流水线：
[Unicode 码点数组]
│
▼
┌─────────────────────────────────┐
│ 第一阶段：字体回退 │
│ FontFallback │
│ - 二分查找 Unicode 区间表 │
│ - 输出每个字符的 font_id │
│ - 时间复杂度 O(log N) │
│ - 零堆分配，SoA 输出 │
└─────────────────────────────────┘
│
▼
┌─────────────────────────────────┐
│ 第二阶段：流式换行 │
│ LineBreakIterator │
│ - 惰性求值，逐行处理 │
│ - O(1) 位掩码禁则判定 │
│ - 方向无关（横排/竖排） │
│ - 不回溯，不预加载 │
└─────────────────────────────────┘
│
▼
┌─────────────────────────────────┐
│ 第三阶段：逐行布局 │
│ LayoutEngineCore │
│ - 联合仲裁（禁则×悬挂×压缩） │
│ - 计算每个字符的精确位置 │
│ - SoA 切片输出 │
│ - 无状态纯函数 │
└─────────────────────────────────┘
│
▼
┌─────────────────────────────────┐
│ 渲染后端（用户自定义） │
│ WebGPU / Canvas2D / SVG │
│ - 从纹理图集中采样字形 │
│ - 一次 DrawCall 渲染万字符 │
│ - 60 FPS 恒定帧率 │
└─────────────────────────────────┘


## 数据流
输入：codepoints: &[u32] (长度 N)
char_widths: &[f32] (长度 N)
line_dim: f32
direction: u8
head_mask: u64
end_mask: u64
policy: LayoutPolicy
fallback: &FontFallback

输出：result_font_ids: &mut [u8] (长度 N)
result_positions: &mut [f32] (长度 N)

返回值：总行数 u32


## 核心特性

| 特性 | 实现方式 | 优势 |
|------|---------|------|
| 零堆分配 | 预分配缓冲区 | 消除 GC 暂停，性能可预测 |
| SoA 数据布局 | 并行数组 | CPU 缓存友好，SIMD 友好 |
| 位掩码禁则 | 64 位位掩码 | O(1) 判定，编译期计算 |
| 惰性求值 | 按需计算下一行 | 内存占用恒定，适合长文本 |
| 方向无关 | direction 旗标 | 横排/竖排共享核心逻辑 |
| 纹理图集 | 字形烘焙 | 一次 DrawCall 渲染万字符 |

## 性能基准
10万字竖排禁则换行：3.47ms
28 字多语言混排： < 1ms
空字符串： < 0.1ms
绘制调用： 1 次 DrawCall


## 设计哲学

1. **数据永不回头**：从输入到输出，数据始终单向流动
2. **决策永不重复**：每个字符的字体归属一次确定，后续直接使用
3. **内存永不泄漏**：所有缓冲区由调用者管理，引擎只做计算
